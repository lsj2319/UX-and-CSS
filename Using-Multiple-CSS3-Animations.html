Using Multiple CSS3 Animations

This is part 4 of a series on CSS3 Animations. This time we will cover how display multiple animations on a screen using CSS3 Animations.

A CSS Animation is not an Island

So far in this series we have only practiced animating objects by themselves. While this is good for some purposes like alerts or mobile navigation display, if we want to start using CSS3 for animating infographics or more complex features will will need to add more than one animation to a page.

  • Wrap animated elements in containers to prevent weird overlap or display.
  • The order in which your animations are declared in the CSS files does not matter, the timing coordinates what happens when.
  • Performance of the page will be affected more by what you are asking the CSS to do than by how many animations you have, but stacking 15 animated sets of charts will have an effect. The Google Developer site has great tips on performance. The most important things to know for now are that using animation to change page layout or to "paint" objects strains performance the most.

Build the HTML Page

First we will create a basic HTML page. Within the body of the page we need to add a text heading, a paragraph, and a div that will act as a button.
<section>
<div>
<h2>Howdy</h2></div>
<div>
<p>Let's have fun with CSS3 Animations, yeah?</p>
</div>
<div class="funglow"><a href="#">Join the fun</a>
</div>
</section>

Set up the CSS

After setting up the HTML we will add attributes and properties to the CSS elements we want to display. Notice that the animation name is declared as a value of the animation or animation-name properties.

h2 {
color: green;
font-family: verdana;
font-weight: 300;
font-size: 36px;
text-align: center;
text-transform: uppercase;
animation-duration: 2s;
animation-name: slideInRight;
-webkit-animation-duration: 2s;
-webkit-animation-name: slideInRight;
margin-top: 100px;
}

/*a simple paragraph but we could easily animate it if we wanted to */

p {
text-align: center;
}

/* setting the link color of the page and removing the underline. For production sites I would make a separate class for links in button shapes or containers. */

a {
text-decoration: none;
color: yellow;
text-align: center;
}

/* next create a class for the buttony rectangle we will be animating. This is quick and dirty and I am doing an evil thing by using absolute positioning for a class. Do not do this on production sites or you will be eaten by a grue. */

.funglow {
position: absolute;
top: 80%;
left: 30%;
align: center;
color: yellow;
width: 200px;
height: 60px;
vertical-align: middle;
line-height: 60px;
background-color: green;
font-family: verdana;
font-weight: 200;
font-size: 20px;
text-align: center;
text-transform: uppercase;
padding: 5px;
animation: 6s colorChange;
-webkit-animation: 6s colorChange;
margin: 0, auto;
}

/*now it is time for some keyframes fun. This is where the magic happens. First I am repeating an example we explored in part 3 of this series. */

@keyframes slideInRight {
from {
margin-left: 100%;
width: 200%;
}
to {
margin-left: 0%;
width: 100%;
}
}

@-webkit-keyframes slideInRight {
from {
margin-left: 100%;
width: 200%;
}
to {
margin-left: 0%;
width: 100%;
}
}

/* Here is a new keyframe called "colorChange". The only property we are changing is the background-color. Remember we specified the animation duration value earlier in the funglow class. */

@keyframes colorChange {
from {
background-color: green;
}
to {
background-color: #6600cc;
}
}

@-webkit-keyframes colorChange {
from {
background-color: green;
}
to {
background-color: #6600cc;
}
}

 

You should see the text fly in and the background color of the button cycle from green to blue and back to blue again.
